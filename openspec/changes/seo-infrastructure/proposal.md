## Why

The portfolio site lacks essential SEO infrastructure: pages shared on social media have no fallback preview image, there is no structured data for search engines, no RSS feed for blog subscribers, no robots.txt to guide crawlers, and blog post reading time is hardcoded instead of calculated from actual content. These are foundational SEO and discoverability features that every content-driven site needs.

## What Changes

- Add a default og:image fallback in `BaseLayout.astro` so every page gets a social preview image even when no explicit image prop is provided
- Add JSON-LD structured data: `BlogPosting` schema on individual blog post pages (`[slug].astro`), `Person` schema on the homepage (`index.astro`)
- Add an RSS feed endpoint via `@astrojs/rss` at `src/pages/rss.xml.ts`
- Add `public/robots.txt` that references the existing sitemap (already generated by `@astrojs/sitemap`)
- Wire the installed `reading-time` package into `[slug].astro` to calculate actual reading time from post content and pass it to `BlogHeader`

## Capabilities

### New Capabilities

- `seo-metadata`: Covers og:image fallback, JSON-LD structured data (BlogPosting and Person schemas), and robots.txt
- `rss-feed`: Covers the RSS feed endpoint configuration and generation

### Modified Capabilities

- `blog-module`: Reading time calculation changes from hardcoded default to computed value passed from the blog post page to BlogHeader

## Impact

- **Files modified**: `src/layouts/BaseLayout.astro`, `src/pages/blog/[slug].astro`, `src/pages/index.astro`, `src/modules/blog/components/BlogHeader.astro` (prop usage only, no interface change needed)
- **Files created**: `src/pages/rss.xml.ts`, `public/robots.txt`
- **Dependencies added**: `@astrojs/rss`
- **Existing dependencies used**: `reading-time` (already in package.json), `@astrojs/sitemap` (already configured)
- **No breaking changes**: All modifications are additive; existing pages continue to work unchanged
